# 🤖 AI Change Reviewer

A stateless Spring Boot backend service that generates AI-powered changelog summaries from GitHub Pull Requests and branch comparisons using the Google Gemini API.

## 📋 Features

- **GitHub Integration**: Fetches commit information and file diffs from GitHub REST API
- **AI-Powered Analysis**: Uses Google Gemini 1.5 Flash to analyze changes intelligently
- **Structured Reports**: Generates markdown reports with:
  - ✨ Features Added
  - 🐛 Bug Fixes
  - ⚡ Improvements
  - 📊 Business Impact
  - ⚠️ Risk Considerations
  - 🧪 Recommended Test Scenarios
- **Simple Web UI**: Vanilla HTML/CSS/JS frontend for easy access
- **Stateless Architecture**: No database, no persistence layer
- **Clean Design**: Implements Strategy, Factory, and Builder patterns

## 🛠️ Tech Stack

- **Language**: Java 17+
- **Framework**: Spring Boot 3.3
- **Build Tool**: Maven
- **External APIs**:
  - GitHub REST API v3
  - Google Gemini API (gemini-1.5-flash model)
- **HTTP Client**: Spring RestTemplate
- **Testing**: JUnit 5, Mockito

## ⚙️ Setup Instructions

### Prerequisites

- Java 17 or higher
- Maven 3.6+
- GitHub account (for API access, optional)
- Google Gemini API key

### 1. Obtain Required API Keys

**GitHub API Token (Optional)**:
- Go to https://github.com/settings/tokens
- Create a new Personal Access Token (Fine-grained tokens recommended)
- Required scopes: `repo` (for private repos), `public_repo` for public repos
- Copy the token

**Google Gemini API Key**:
- Go to https://ai.google.dev
- Click "Get API Key" or go to Google Cloud Console
- Create a new API key
- Copy the key

### 2. Clone and Build

```bash
git clone https://github.com/aman0710/ai-change-reviewer.git
cd ai-change-reviewer
mvn clean install
```

### 3. Configure Environment Variables

Set the required environment variable:

**On Linux/Mac**:
```bash
export GEMINI_API_KEY="your-gemini-api-key"
```

**On Windows (PowerShell)**:
```powershell
$env:GEMINI_API_KEY = "your-gemini-api-key"
```

### 4. Run the Application

```bash
mvn spring-boot:run
```

The application will start on `http://localhost:8080`

### Optional Configuration

You can customize settings in `src/main/resources/application.properties`:

```properties
spring.application.name=ai-change-reviewer
server.port=8080

# GitHub API Base URL (default: https://api.github.com)
github.api.base-url=https://api.github.com

# Gemini API Configuration
gemini.api.key=${GEMINI_API_KEY}
gemini.api.url=https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent
```

## 🚀 Usage

### Web UI

1. Open `http://localhost:8080` in your browser
2. Enter a GitHub URL:
   - **Pull Request**: `https://github.com/owner/repo/pull/123`
   - **Branch Compare**: `https://github.com/owner/repo/compare/main...feature-branch`
3. (Optional) Enter GitHub token for private repositories
4. Click "Generate Changelog"
5. The markdown file will automatically download

### API Usage

**Endpoint**: `POST /api/changelog/generate`

**Request**:
```bash
curl -X POST http://localhost:8080/api/changelog/generate \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://github.com/owner/repo/pull/123",
    "githubToken": "optional-token-for-private-repos"
  }'
```

**Response** (200 OK):
```
Content-Type: text/markdown
Content-Disposition: attachment; filename="changelog-repo-20240101-120000.md"
```

**Error Responses**:

- `400 Bad Request`: Invalid URL format
  ```json
  {
    "error": "Invalid URL",
    "message": "Invalid GitHub URL format. Supported formats: https://github.com/{owner}/{repo}/pull/{number} or https://github.com/{owner}/{repo}/compare/{base}...{head}"
  }
  ```

- `404 Not Found`: Repository or PR not found
  ```json
  {
    "error": "GitHub API Error",
    "message": "Failed to fetch commits from GitHub: 404 Not Found"
  }
  ```

- `502 Bad Gateway`: GitHub or Gemini API failure
  ```json
  {
    "error": "AI Service Error",

  }
  ```

## 🏗️ Architecture

### Design Patterns

#### 1. **Strategy Pattern** - GitHub Fetch Logic
Different URL types (PR vs branch compare) require different API calls:

```java
public interface GitHubFetchStrategy {
    List<CommitInfo> fetchCommits(GitHubContext context, String token);
    List<FileChange> fetchFileChanges(GitHubContext context, String token);
}
```

- `PullRequestFetchStrategy`: Uses GitHub PR endpoints
- `BranchCompareFetchStrategy`: Uses GitHub compare endpoint

#### 2. **Factory Pattern** - Strategy Resolution
Factory resolves the correct strategy based on URL type:

```java
public class GitHubFetchStrategyFactory {
    public GitHubFetchStrategy resolve(GitHubUrlType type);
}
```

#### 3. **Builder Pattern** - Report Construction
Fluent builder for constructing markdown reports:

```java
String report = new ChangeReportBuilder()
    .addHeader(owner, repo, url, commitCount)
    .addFeatures(features)
    .addBugFixes(bugFixes)
    // ... more sections
    .build();
```

## 🔐 Security Considerations

- **API Keys**: Store API keys as environment variables, never in code
- **HTTPS Only**: All external API calls use HTTPS
- **Input Validation**: All URLs are validated before API calls
- **Error Handling**: Error messages don't leak sensitive information
- **Token Handling**: GitHub tokens are passed only when explicitly provided

## 🐛 Known Limitations

- Maximum diff size per file: 50,000 characters (truncated for Gemini API)
- GitHub API rate limiting applies (60 requests/hour unauthenticated, 5000/hour authenticated)
- Large pull requests (100+ files) may take longer to process

## 🚫 Out of Scope

- No database or persistence
- No message queue support
- No authentication/authorization
- No caching layer
- GitLab/Bitbucket not supported
